Hunter-Killer

by: Jonas Brown

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

When I was younger, I loved playing an arcade game called "Sea Wolf". I tried to keep some of the spirit of that game in mine, despite the numerous differences between the two. I decided to keep the idea of firing torpedoes at ships moving across the top of the screen, but decided I wanted to design the rest of the game from scratch, creating a new scoring scheme and adding friendly ships.

2. What I Learned

Testers suggested that I have the ships speed up over time.

3. Architecture of My Game

I have a runner class to run the game, a GamePanel class to control the game states and background displays, and an ObjectManager class to control which game objects are active. I have classes controlling the look and behavior of all 4 types of ships, allied destroyers, allied carriers, enemy destroyers, and enemy carriers. I also have a class to control the torpedoes, and a Crosshair class to manage the player-controlled crosshair. The crosshair class determines the generation and placement of torpedoes when they are launched. The ObjectManager class determines which ships and torpedoes need to be deactivated and removes them. The GamePanel controls all of the classes except the runner in order to activate the game at the appropriate time. The runner runs the game, using the GamePanel.

4. Technical Implementation

In particular, I am proud of the ship hierarchy I created for the game. I created an overarching "Ship" class, which I used as a framework. I then created "Allied Ship" and "Enemy Ship" classes, and from those, a "Destroyer" and "Carrier" type of each. This allowed me to give the four ship types inherited code which I could use to simplify their creation.

5. Most Challenging Part

The most challenging part of the project was adding an Instructions game state near the end of development. I decided not to use a message dialog alone to display instructions, as allied ships needed to be distinguished from enemy ships. I created the Instructions game state so that instructions appeared in message dialogs, but there was an image showing players who to shoot at as well. This game state, which was to be reached with the space bar, also made me realize that I was using the space bar as the fire button, and I therefore had to change the fire button to the up arrow key.

6. Future Fixes and Updates

If I were to continue working on the game, I would like to add more ship classes with their own appearances and attributes. I would also like to adapt the instructions into more visually appealing screens instead of message dialogs.
